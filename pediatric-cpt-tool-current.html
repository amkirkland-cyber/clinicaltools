How to manually update JSON

The tool stores its data inside this HTML file in a script tag:

<script id="app-data" type="application/json">
{
  "wellVisits": [],
  "acuteEM": [],
  "poct": []
}
</script>

Basic rules

  • Use double quotes around all keys and text values.
  • Separate items in a list with commas.
  • Do not put a comma after the last item in a list or object.
  • Each well visit has a label, visit codes, and sections.

Example: add an in-office code to a well visit

{
  "label": "9 mo",
  "visitCodes": { "new": "99381", "established": "99391" },
  "sections": {
    "admin": ["Capillary blood collection: 36416 x1"],
    "inOffice": ["Hemoglobin POCT: 85018 x1"],
    "vaccineProducts": [],
    "seasonalProducts": ["90686 Fluzone"],
    "seasonalAdmin": ["Flu admin: 90460 x1"],
    "externalLab": []
  }
}

Section names the tool understands

Section keyMeaning
visitVisit code line, usually generated automatically from new/established choice.
adminRoutine vaccine administration codes.
inOfficeExtra in-office billable codes, such as 85018, 36415, 36416, 99174.
vaccineProductsRoutine vaccine product CPT codes for reference.
seasonalProductsSeasonal/conditional products, such as Fluzone or Beyfortus.
seasonalAdminSeasonal/conditional admin codes, such as flu admin or Beyfortus admin.
externalLabExternal lab codes not billed from your office.
referenceGeneral reference-only codes.

Example: add a new POCT reference

{
  "test": "Glucose, blood by monitoring device",
  "code": "82962",
  "description": "POCT fingerstick/capillary glucose"
}
